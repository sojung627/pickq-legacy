[채용 관점 어필 포인트]

1) 도메인 난이도: 역경매형 거래 모델 직접 설계/구현
- 일반 중고거래와 다른 "구매요청 -> 다중 입찰 -> 낙찰자 선정" 흐름을 상태머신으로 구현.
- auction_status / bid_status / order_status를 분리해 복잡한 전이를 관리.
- 낙찰 시 주문 생성, 결제/배송/구매확정으로 이어지는 트랜잭션형 시나리오를 완성.

2) 실시간 기능 경험: WebSocket + STOMP + SockJS
- 채팅: 채팅방 단위 topic 브로드캐스트(/topic/chatroom/{id}).
- 알림: 사용자 단위 topic 브로드캐스트(/topic/notifications/{memIdx}).
- HTTP API + 실시간 채널을 병행해 UX 응답성을 높임.

3) 데이터 모델링 역량
- 코드 테이블(role/grade/status/category/type) 분리로 확장성과 무결성 확보.
- CHECK/UNIQUE/FK/ON DELETE 정책을 기능 의도에 맞게 적용.
- 거래 마스터(orders) + 세부(payment/delivery) 분리 설계로 이력 추적과 유지보수성 강화.

4) 실무형 백엔드 구조
- Spring Boot MVC + MyBatis XML Mapper로 SQL 가시성과 튜닝 여지 확보.
- 서비스/컨트롤러 레이어 분리, 세션 기반 인증/인가, 인터셉터 리다이렉트 처리 구현.
- 파일 업로드, 외부 SDK(문자 인증), 결제 API 연동 등 실서비스 요소를 통합.

5) 커뮤니티/거래 융합형 서비스 구현
- 게시판/댓글/좋아요 + 거래/리뷰/프로필이 단절되지 않고 연결된 사용자 경험 제공.
- 마이페이지에서 구매/판매/경매/입찰/게시글/리뷰를 통합 관리.

6) 운영 안정성 포인트
- 로그인 실패 횟수 제한/잠금 로직(브루트포스 완화).
- 커스텀 에러 페이지와 예외 노출 최소화 설정.
- soft delete 기반 데이터 보전 전략.

7) 협업 역량(포트폴리오 강조 문구 예시)
- Git Flow 기반 브랜치 전략 및 병합 경험.
- 이슈/프로젝트 도구를 사용한 스프린트형 개발.
- 기획-설계-구현-테스트-발표까지 전 주기 참여 경험.
