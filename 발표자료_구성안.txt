[PickQ 발표자료 구성안]
권장 발표 시간: 12~15분
권장 장수: 14장

1. 타이틀
- 핵심 문장: "PickQ는 스포츠 용품 거래를 역경매 방식으로 재해석한 웹 플랫폼입니다."
- 포함 내용: 프로젝트명, 팀명, 발표자, 기간
- 시각 자료: 서비스 로고 + 메인 화면 캡처

2. 문제 정의와 기획 배경
- 핵심 문장: "기존 중고거래는 판매자 중심이어서 구매자 요구를 반영하기 어렵습니다."
- 포함 내용: 기존 방식의 한계, 역경매 도입 이유
- 시각 자료: 기존 거래 vs 역경매 비교 다이어그램

3. 서비스 한 줄 가치 제안
- 핵심 문장: "구매자가 요청하고 판매자가 제안하는 구조로, 조건 맞춤형 거래를 만든다."
- 포함 내용: 타깃 사용자, 차별 포인트 3가지
- 시각 자료: 가치 제안 카드 3개

4. 주요 기능 요약
- 핵심 문장: "회원/역경매/입찰/거래/커뮤니티/실시간 기능이 하나의 흐름으로 연결됩니다."
- 포함 내용: 기능 맵(회원, 경매, 입찰, 결제, 리뷰, 채팅, 알림)
- 시각 자료: 기능 맵 아이콘형

5. 사용자 플로우(End-to-End)
- 핵심 문장: "요청 등록부터 구매확정/리뷰까지 거래 전 주기를 구현했습니다."
- 포함 내용: 구매자/판매자 양방향 플로우
- 시각 자료: Swimlane(구매자/판매자)

6. 기술 스택/개발 환경
- 핵심 문장: "Spring Boot + MyBatis + MySQL + WebSocket 기반의 실무형 조합입니다."
- 포함 내용: 백엔드, DB, 프론트, 협업 도구
- 시각 자료: 스택 로고 그리드
- 근거 파일: 환경.txt

7. 시스템 아키텍처
- 핵심 문장: "MVC 기반 서버 렌더링과 STOMP 실시간 채널을 결합해 반응성과 안정성을 동시에 확보했습니다."
- 포함 내용: 브라우저-컨트롤러-서비스-MyBatis-MySQL, WebSocket 브로커 경로
- 시각 자료: 아키텍처 블록 다이어그램

8. DB 설계 철학
- 핵심 문장: "코드 테이블 분리 + 강한 제약조건으로 무결성과 확장성을 확보했습니다."
- 포함 내용: role/grade/status/category/type 분리 이유, CHECK/UNIQUE/FK 설계
- 시각 자료: 핵심 테이블군 도식
- 근거 파일: DB분석.txt

9. 핵심 도메인 모델(역경매-입찰-거래)
- 핵심 문장: "auction-bid-orders-payment-delivery-review가 상태 전이로 연결됩니다."
- 포함 내용: 상태값(open/closed, normal/won 등), 전이 시점
- 시각 자료: 상태 전이 다이어그램

10. API 설계 규모
- 핵심 문장: "Controller 전체 엔드포인트를 표준화해 기능 누락 없이 관리했습니다."
- 포함 내용: REST + STOMP 매핑, 기능군별 분포
- 시각 자료: 기능군별 API 개수 막대그래프
- 근거 파일: api 명세서.txt

11. 실시간 기능 구현 포인트
- 핵심 문장: "채팅과 알림을 topic 단위로 분리해 사용자 경험을 즉시화했습니다."
- 포함 내용: /app/chat/send -> /topic/chatroom/{id}, /app/notifications/ping -> /topic/notifications/{memIdx}
- 시각 자료: 메시지 라우팅 흐름도

12. 운영 안정성/보안 대응
- 핵심 문장: "데모 수준을 넘어 운영 리스크를 고려한 방어 로직을 적용했습니다."
- 포함 내용: 로그인 실패 잠금, 세션 관리, 커스텀 에러 페이지, soft delete
- 시각 자료: 리스크-대응 매트릭스

13. 성과와 회고
- 핵심 문장: "복잡한 상태 기반 도메인을 팀 협업으로 통합 완료했습니다."
- 포함 내용: 완료 범위, 배운 점, 기술적 챌린지 해결 사례
- 시각 자료: Before/After 카드

14. 채용 어필 포인트 / 마무리
- 핵심 문장: "CRUD를 넘어 상태 전이형 거래와 실시간 커뮤니케이션을 완성한 경험이 핵심 강점입니다."
- 포함 내용: 아키텍처/도메인/협업 관점 강점 3줄
- 시각 자료: 키워드 하이라이트
- 근거 파일: 어필포인트.txt

[발표 시간 배분(15분 기준)]
- 1~4장: 3분
- 5~9장: 6분
- 10~12장: 3분
- 13~14장: 2분
- 질의응답 유도: 1분

[발표 톤 가이드]
- 기능 나열보다 "문제 -> 설계 선택 -> 결과" 순서로 설명.
- 화면 데모와 기술 슬라이드를 번갈아 배치해 집중도 유지.
- 숫자/제약조건/상태값 같은 구체성을 반드시 포함.
