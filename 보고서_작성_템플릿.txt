[최종 보고서 작성 템플릿]
문서 목적: 성과 발표회 제출 + 포트폴리오 첨부용
권장 분량: 15~25페이지(텍스트 문서 기준)

1. 프로젝트 개요
- 프로젝트명 / 기간 / 팀 구성
- 서비스 목적
- 핵심 문제 정의

2. 요구사항 및 범위
- MVP 범위
- 최종 구현 범위
- 제외/보류 범위

3. 개발 환경 및 기술 스택
- 환경/언어/프레임워크/DB/협업도구 정리
- 선택 이유(왜 이 조합인지)
- 참고 파일: 환경.txt

4. 시스템 아키텍처
- 전체 구성도(클라이언트-서버-DB-실시간)
- 요청/응답 흐름
- 세션 인증/인터셉터 흐름

5. DB 설계 분석
- 테이블 그룹(회원/옥션/거래/커뮤니티/채팅/알림)
- 핵심 FK 관계
- 무결성 제약(UNIQUE, CHECK, ON DELETE)
- 참고 파일: DB분석.txt

6. 핵심 기능 설계 및 구현
- 회원/인증
- 역경매/입찰
- 거래/결제/배송/리뷰
- 커뮤니티
- 실시간 채팅/알림
- 각 기능마다: 문제 -> 설계 -> 구현 결과 -> 화면 캡처

7. API 명세 요약
- 기능군별 대표 API 표
- 엔드포인트 개수/분포
- STOMP 매핑 포함
- 참고 파일: api 명세서.txt

8. 트러블슈팅
- 사례 1: 상태 전이 충돌 또는 예외 처리
- 사례 2: 프론트/백엔드 연동 불일치
- 사례 3: 소켓/알림 동기화 이슈
- 항목 형식: 증상 -> 원인 -> 해결 -> 재발 방지

9. 테스트 및 검증
- 시나리오 기반 수동 테스트 항목
- 데모 체크리스트
- 남은 리스크와 개선 계획

10. 성과 및 회고
- 정량/정성 성과
- 협업 방식(Git Flow, 이슈관리)
- 개인 기여도 및 성장 포인트

11. 채용 어필 포인트
- 도메인 복잡도 해결 경험
- 실시간 기능 설계 경험
- 데이터 무결성 중심 설계
- 참고 파일: 어필포인트.txt

12. 향후 고도화 계획
- 테스트 자동화 강화
- 성능/모니터링 고도화
- 알림/결제 이벤트 아키텍처 개선
