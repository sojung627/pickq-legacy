[PickQ 2인 시연 초압축 큐시트 - 1페이지]
총 8분 목표 (최대 9분)
구성: A(구매자, 왼쪽 화면) / B(판매자, 모바일 오른쪽 화면)

==================================================
0. 시작 전 20초 체크
==================================================
- A/B 모두 로그인 완료
- 같은 경매 목록 화면 대기
- 채팅/알림 아이콘 노출 확인
- 타이머 시작

==================================================
1. 오프닝 (0:00~0:20)
==================================================
- A 대사
: "왼쪽은 구매자, 오른쪽은 판매자 모바일 화면입니다."
: "요청 생성부터 거래완료/리뷰까지 1사이클을 8분 내에 보여드리겠습니다."
- B 액션
: 대기

==================================================
2. 구매요청 생성 (0:20~1:30)
==================================================
- A 액션
1) 경매 등록 진입
2) 카테고리/희망가/설명 입력 후 등록
3) 생성된 경매 상세 진입
- A 대사
: "거래 시작점이 판매글이 아니라 구매요청입니다."
: "생성 즉시 진행중 상태로 노출됩니다."
- B 액션
: 같은 경매 검색 준비

==================================================
3. 판매자 입찰(모바일 강조) (1:30~2:40)
==================================================
- B 액션
1) 모바일에서 해당 경매 진입
2) 입찰 등록(이미지+가격+메시지)
- B 대사
: "모바일에서도 핵심 입력 동선이 동일하게 동작합니다."
- A 액션
: 경매 상세에서 입찰 반영 확인
- A 대사
: "입찰이 즉시 누적되어 구매자가 비교할 수 있습니다."

==================================================
4. 실시간 채팅/알림 (2:40~3:40)
==================================================
- A 액션
: 채팅 1건 전송
- B 액션
: 즉시 답장 1건 전송
- A/B 공통 확인
: 알림 또는 읽지않음 수치 변화 확인
- A 대사
: "새로고침 없이 양측 화면이 즉시 동기화됩니다."

==================================================
5. 낙찰 및 거래 진입 (3:40~4:40)
==================================================
- A 액션
1) 낙찰자 선정 클릭
2) 주문 내역 화면 진입
- A 대사
: "낙찰은 상태 변경을 넘어서 주문 라이프사이클 시작 이벤트입니다."
- B 액션
: 판매자 주문/판매 내역 화면 대기

==================================================
6. 배송 처리 -> 구매확정 (4:40~6:10)
==================================================
- B 액션
1) 배송 처리(운송장/발송)
- A 액션
2) 구매확정 처리
- A 대사
: "orders-payment-delivery-confirm 흐름으로 엔드투엔드가 닫힙니다."
- B 대사
: "판매자/구매자 화면의 상태가 일관되게 변경됩니다."

==================================================
7. 리뷰/평판 반영 (6:10~7:00)
==================================================
- A 액션
1) 리뷰 작성
2) 프로필 또는 리뷰 상세에서 반영 확인
- A 대사
: "거래 종료 데이터가 사용자 평판으로 연결됩니다."

==================================================
8. 마무리 3문장 (7:00~7:30)
==================================================
- A 대사
1) "역경매 핵심 사이클을 한 번에 완주했습니다."
2) "모바일 반응형과 실시간 웹소켓 동작을 동시에 검증했습니다."
3) "상태 전이형 거래 로직을 실제 사용자 경험으로 연결한 것이 핵심 성과입니다."

==================================================
9. 늘어짐 방지 규칙(현장용)
==================================================
- 입력 화면 10초 이상 머물지 않기
- 설명은 액션 전 1문장, 결과 후 1문장만
- 클릭 3번 넘어가면 즉시 준비된 화면으로 점프
- 장애 발생 시 15초 내 플랜B 전환

==================================================

==================================================

: "결제 연동 구간은 라우팅만 확인하고 주문 상태 전이로 바로 이어가겠습니다."
- 소켓 지연
: "실시간 구간은 REST 조회 화면으로 동일 결과를 바로 확인하겠습니다."
- 데이터 꼬임
: "사전 준비한 데모 경매 번호로 전환해 핵심 흐름을 계속 진행하겠습니다."
